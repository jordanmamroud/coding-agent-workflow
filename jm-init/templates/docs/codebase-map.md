# Codebase Map

Where things live and why. **Read at the start of any non-trivial task** to know where things are without grepping the whole tree.

**Update this file when:**
- Adding a new route folder under `app/`
- Adding a significant `_lib/` file (real logic, not a small formatter)
- Adding or removing tables in `src/db/schema.ts`
- Renaming or moving anything in this map

Keep it at navigation altitude. Don't list every component file — that's noise. Do list every route folder, every `actions.ts`, every `src/` file, and any `_lib/` file with real logic.
